Five Major Features

  1. Real-Time Vital Signs Display - Display current patient vital signs including oxygen saturation, heart rate, pulse, blood sugar level, and blood pressure on the device screen with automatic refresh capabilities.
  2. Alert and Notification System - Generate audible and visual alerts when vital signs fall outside safe ranges, with automatic notifications sent to nurse station computers including patient and room details.
  3. Device Security and Access Control - Provide physical button lock functionality to restrict device usage to authorized personnel only, preventing unauthorized changes to settings or patient data.
  4. Historical Data Management - Store and retrieve vital signs history for configurable time periods, allowing healthcare providers to track patient trends and make informed medical decisions.
  5. Patient Profile Management - Maintain patient information including name, room number, medical history, and custom threshold settings for vital sign alerts based on individual patient needs.

1. Use Case Diagram

Description

This use case diagram illustrates all actors in the Vital Signs Monitoring System and their interactions with various system functionalities. The diagram shows both human actors (Nurse, Physician, Medical Technician) and system actors (Nurse Station System, Data Storage System).

Key Actors and Their Interactions:

  • Nurse: Views vital signs, acknowledges alerts, locks/unlocks device buttons, views patient history
  • Physician: Views vital signs, configures alert thresholds, views history, generates reports, manages patient profiles
  • Medical Technician: Views vital signs, locks/unlocks device buttons
  • Nurse Station System: Receives alert notifications from the monitoring device
  • Data Storage System: Stores and retrieves vital signs data and historical information

2. Sequence Diagram: Alert Generation and Notification

Scenario

This sequence diagram illustrates the complete workflow when a patient's oxygen saturation drops below the safe threshold (90%). It shows the interactions between the sensor, monitoring device, alert manager, alarm system, nurse station, and database.

Key Steps:

  1. Sensor reads oxygen level (88%)
  2. Monitoring device detects value below threshold (90%)
  3. Alert Manager triggers alarm at 85 decibels
  4. Notification sent to Nurse Station with patient details (Room 305, John Doe, O2: 88%)
  5. Alert event logged to database with timestamp and patient information
  6. Nurse acknowledges alert from station
  7. Alarm stops and status updated in database

3. Sequence Diagram: Device Button Lock/Unlock

Scenario

This sequence diagram shows the security workflow when a physician locks the device buttons to prevent unauthorized access. It includes authentication, button disabling, logging, and handling of unauthorized access attempts.

Key Steps:

  1. Physician presses lock button on device
  2. System requests authentication
  3. Authentication system verifies physician credentials from database
  4. Upon successful authentication, button controller disables physical buttons
  5. Lock event logged to database with user ID, timestamp, and device ID
  6. Device displays "Buttons Locked" confirmation message
  7. When physical button is pressed, system detects the attempt
  8. Button controller checks lock status with database
  9. System displays warning message: "Device Locked - Authentication Required"
